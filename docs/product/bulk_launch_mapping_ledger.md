# Bulk Launch Mapping Ledger

Date: 2026-07-05

## Product Slice

The second offline implementation slice turns the first launch workspace into a **Bulk Launch Mapping Ledger**.

The ledger is still local and dry-run only. It does not call Meta APIs, verify real Meta object IDs, load credentials, mutate ad accounts, publish ads, or claim Marketing API compatibility.

## What Changed

- Added optional v2 manifest fields for account aliases, objectives, placements, asset hashes, variant groups, hooks, language, country, UTMs, post references, source lineage, reviewer, and approval timestamp.
- Added `fixtures/fake_agency_creatives/manifest_v2.csv`, generated by `scripts/enrich_fixture_v2.py`.
- Added `fixtures/fake_agency_creatives/asset_metadata.csv`, generated by `scripts/enrich_fixture_v2.py`, so asset validation can use synthetic metadata instead of pretending path strings are enough.
- Added `offline_launch_plan.v2` metadata to exports:
  - `contract_status: local_adapter_draft`
  - `mutation_allowed: false`
  - `meta_api_compatibility: not_claimed`
  - `source_manifest_sha256`
  - `data_classification`
- Enriched every dry-run ad with:
  - `account_mapping`
  - `placement_mapping`
  - `creative`
  - `tracking`
  - `post_lineage`
  - `source_lineage`
  - `approval_record`
  - `operation_contract`
  - `idempotency`
  - `preflight`
- Added validators for unsupported objectives, placement-format mismatch, UTM mismatch, partial UTM mapping, UTM lineage, post-reference intent, incomplete source lineage, reviewer handoff, and idempotency collisions.
- Expanded the static HTML table with account, placement, UTM status, post ID, and source columns.
- Added a `workspace_review_state.v1` export path so mapped rows can carry local review decisions, role ownership, and audit seed into the next backend slice.
- Added `workspace_html_static_audit.v1` output so mapped workspace changes can be checked for required accessibility/workflow affordances without a browser runtime.
- Added `meta_platform_payload_preview.v1` output so mapped rows can be reviewed against Meta-shaped draft payload sequences without API calls.

## Rehearsal Result

Command:

```bash
python3 scripts/enrich_fixture_v2.py
python3 -m meta_importer.cli plan fixtures/fake_agency_creatives/manifest_v2.csv --out runs/fake_agency_creatives_v2/launch_plan.json --review runs/fake_agency_creatives_v2/review_packet.md --html runs/fake_agency_creatives_v2/workspace.html --html-audit runs/fake_agency_creatives_v2/workspace_audit.json --state runs/fake_agency_creatives_v2/review_state.json --platform-preview runs/fake_agency_creatives_v2/platform_preview.json --store-dir runs/fake_agency_creatives_v2/store --asset-metadata fixtures/fake_agency_creatives/asset_metadata.csv
```

Result:

- 100 rows loaded.
- 30 launch-ready candidates.
- 10 needs-review rows.
- 60 blocked rows.
- 100 account aliases.
- 100 placement mappings.
- 100 UTM mappings.
- 100 source-lineage mappings.
- 100 post decisions.
- 10 existing post references.
- 100 local review-state rows.
- 100 asset metadata-backed validation rows.
- Static workspace audit status `pass`.
- 100 platform payload previews, all blocked and validate-only.

## Boundary

This is an offline adapter-ready contract, not a Meta API adapter. Live Meta integration requires separate Platform, Security, QA, and HITL gates.
